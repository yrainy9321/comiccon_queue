# 微信小程序消息模板配置指南

## 概述

本系统支持3种排队消息推送:
1. **提前5位提醒** - 当用户前面还有5个人时发送
2. **叫号通知** - 当轮到用户时发送
3. **过号提醒** - 当用户过号时发送

## 配置步骤

### 1. 在微信公众平台创建订阅消息模板

登录 [微信公众平台](https://mp.weixin.qq.com/) -> 功能 -> 订阅消息 -> 公共模板库

#### 模板1: 提前5位提醒

搜索并添加适合的模板,推荐字段:
- 活动名称 (thing1)
- 前面人数 (number2)
- 排队号码 (number3)
- 提醒时间 (time4)

记录模板ID,例如: `ABC123xyz456`

#### 模板2: 叫号通知

搜索并添加适合的模板,推荐字段:
- 活动名称 (thing1)
- 排队号码 (number2)
- 叫号时间 (time3)

记录模板ID,例如: `DEF789uvw012`

#### 模板3: 过号提醒

搜索并添加适合的模板,推荐字段:
- 活动名称 (thing1)
- 排队号码 (number2)
- 过号时间 (time3)
- 提示信息 (thing4)

记录模板ID,例如: `GHI345rst678`

### 2. 配置环境变量

复制 `.env.example` 为 `.env`:

```bash
cd backend
cp .env.example .env
```

编辑 `.env` 文件,填入你的配置:

```env
# 微信小程序配置
WECHAT_APPID=你的小程序AppID
WECHAT_SECRET=你的小程序AppSecret

# 微信小程序消息模板ID
WECHAT_TEMPLATE_REMINDER_5=你的提前5位提醒模板ID
WECHAT_TEMPLATE_CALLED=你的叫号通知模板ID
WECHAT_TEMPLATE_MISSED=你的过号提醒模板ID

# 服务器配置
PORT=3000
```

### 3. 重启服务器

```bash
npm start
```

## 前端订阅消息授权

用户需要在小程序中授权接收订阅消息。在绑定手环页面或状态页面添加订阅按钮:

```javascript
// 请求订阅消息授权
wx.requestSubscribeMessage({
  tmplIds: ['模板ID1', '模板ID2', '模板ID3'],
  success(res) {
    console.log('订阅结果:', res);
  },
  fail(err) {
    console.error('订阅失败:', err);
  }
});
```

## 测试流程

1. **启动服务器**
   ```bash
   cd backend
   npm start
   ```

2. **打开微信开发者工具**,导入小程序项目

3. **登录小程序**,获取openid和token

4. **绑定手环**
   - 选择活动
   - 输入手环编号
   - 点击绑定
   - 系统会自动将手环绑定到当前微信用户

5. **工作人员叫号**
   - 使用管理员账号登录后台
   - 进入叫号页面
   - 输入号码进行叫号

6. **验证消息推送**
   - 检查微信是否收到订阅消息
   - 查看服务器控制台日志

## 常见问题

### 1. 收不到消息

- 检查用户是否已授权订阅消息
- 检查模板ID是否正确配置
- 检查手环是否正确绑定到微信用户
- 查看服务器日志确认消息是否发送成功

### 2. 消息发送失败

常见错误码:
- `40037`: 模板ID不正确
- `40003`: openid不正确或用户未关注
- `47003`: 模板参数格式错误
- `48001`: 没有权限使用该模板

### 3. 如何查看日志

服务器会在控制台输出:
```
手环 XXX 已绑定到微信用户 YYY
发送叫号通知: 手环=XXX, 号码=ZZZ
消息发送成功: openid=YYY, template=TEMPLATE_ID
```

## 技术实现

### 后端API

- `POST /api/wechat/login` - 微信登录获取openid
- `POST /api/wechat/bind-bracelet` - 绑定手环到微信用户
- `POST /api/queue/call` - 叫号(自动触发消息推送)

### 消息推送逻辑

1. 用户绑定手环时,保存 `braceletId -> openid` 映射
2. 叫号时,查找对应手环的openid
3. 根据情况发送不同类型的消息:
   - 正常叫号: 发送叫号通知
   - 过号: 发送过号提醒
4. 更新当前号码后,检查是否有用户需要"提前5位"提醒

### 数据结构

```json
// bracelet_bindings.json
{
  "BRACELET001": {
    "openid": "oXXXXX...",
    "braceletId": "BRACELET001",
    "boundAt": "2026-04-27T10:00:00.000Z"
  }
}
```

## 注意事项

1. **订阅消息需要用户主动授权**,每个模板ID每次授权只能发送一次消息
2. **长期订阅消息**需要申请特定类目,建议使用一次性订阅
3. **测试环境**可以使用开发版/体验版的消息推送
4. **生产环境**需要确保小程序已发布且模板已审核通过
