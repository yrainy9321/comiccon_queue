# 微信登录故障排查指南

## 🔍 常见问题

### 问题1：提示"初始化失败"

**可能原因：**
1. 后端服务未启动
2. API地址配置错误
3. 网络连接问题
4. 微信小程序域名未配置

**解决方案：**

#### 方案1：检查后端服务
```bash
# 检查后端是否运行
lsof -i:3000

# 如果没有运行，启动后端
cd /Users/bawangchaji/Documents/课程/kaifa/comiccon-queue
npm start
```

#### 方案2：检查API地址
在 `frontend/app.js` 中确认：
```javascript
globalData: {
  API_URL: 'http://localhost:3000/api'  // 开发环境
}
```

**注意**：
- 开发者工具中使用 `localhost`
- 真机测试需要使用局域网IP或已配置的域名

#### 方案3：关闭域名校验（开发阶段）
在微信开发者工具中：
1. 点击右上角「详情」
2. 选择「本地设置」
3. ✅ 勾选「不校验合法域名、web-view（业务域名）、TLS版本以及HTTPS证书」

#### 方案4：查看控制台日志
打开开发者工具的Console面板，查看：
```
App launched
开始微信登录流程...
获取到code: 071xxxxx
```

如果看到错误信息，根据具体错误进行排查。

### 问题2：登录后显示"临时模式"

**说明：**
这是正常的降级机制。当后端服务不可用时，系统会自动使用临时ID，保证用户可以继续使用基本功能。

**红色徽章含义：**
- ⚠ 图标：表示警告状态
- "临时模式"：表示使用的是临时ID
- "后端服务未连接，功能受限"：提示当前状态

**如何恢复正常：**
1. 启动后端服务：`npm start`
2. 清除缓存后重新编译小程序
3. 或者点击页面的刷新按钮（如果有）

---

### 问题3：wx.request 请求失败

**错误信息示例：**
```
request:fail url not in domain list
```

**解决方案：**

#### 开发环境
在开发者工具中关闭域名校验（见方案3）

#### 生产环境
需要在微信公众平台配置服务器域名：
1. 登录 https://mp.weixin.qq.com/
2. 进入「开发」→「开发管理」→「开发设置」
3. 找到「服务器域名」
4. 添加你的后端地址到「request合法域名」

---

### 问题4：获取不到code

**可能原因：**
- 微信登录接口调用失败
- 小程序配置问题

**解决方案：**
1. 检查AppID是否正确
2. 确认小程序已发布或在测试名单中
3. 查看控制台是否有 `wx.login失败` 的错误信息

---

### 问题5：后端返回 "invalid appid"

**原因：**
backend/.env 中配置的AppID不正确或未配置

**解决方案：**
1. 创建 backend/.env 文件
2. 填入正确的AppID和Secret：
```env
WECHAT_APPID=wx1234567890abcdef
WECHAT_SECRET=your_secret_here
```
3. 重启后端服务

## 📊 正常流程 vs 异常流程

### 正常流程（后端可用）
```
打开小程序
  ↓
显示"正在登录..."
  ↓
调用 wx.login() 获取 code
  ↓
发送 code 到后端
  ↓
后端返回 openid + token
  ↓
显示"✓ 已登录"（紫色徽章）
  ↓
可以正常使用所有功能
```

### 降级流程（后端不可用）
```
打开小程序
  ↓
显示"正在登录..."
  ↓
调用 wx.login() 获取 code
  ↓
发送 code 到后端 → 失败
  ↓
等待5秒超时
  ↓
生成临时ID
  ↓
显示"⚠ 临时模式"（红色徽章）
  ↓
可以使用基本功能（扫码等）
  ↓
但无法接收消息推送
```
